리포트는 결과물,
스킬은 그 도구다.

The report is the output. The skill is the tool that makes it — and you can install it.

SNAP-SCOUT은 한 명의 작가를 위한 작은 도구로 시작했지만, 만드는 과정에서 다른 작가들에게도 그대로 쓰일 수 있는 형태가 됐다.

이 페이지는 그 스킬이 어떻게 만들어졌는지, 그리고 다른 사람이 자기 Claude Code에 어떻게 설치해서 쓸 수 있는지를 정리한다.

01

어떻게 만들어졌는가

The making — from one photo to a reusable skill.
DAY 0 → DAY 1
SINGLE SESSION

처음엔 그냥 한 장의 사진이었다. 스냅 작가 채빈이 핀터레스트에서 발견한 홍콩의 어느 계단. "여기 어디 같아?" 라는 질문에서 시작했다.

"레퍼런스 사진을 찾으면 거기 가서 찍어보고 싶은데, 매번 어디인지 알아내는 데만 한참 걸려요." — 채빈, 스냅 작가

한 번 해결하고 나면 같은 일을 반복하지 않기를 바랐다. 한자 OCR, 비주얼 시그니처, 핀터레스트 알고리즘이 끌어오는 비슷한 사진들 — 이 흐름 자체를 명령 한 줄로 압축할 수 있다면, 다음 사진도 같은 방식으로 처리될 수 있다.

그래서 두 가지를 만들었다. 하나는 리포트 그 자체 (이 컬렉션의 №001), 다른 하나는 그 리포트를 또 만들 수 있는 Claude Skill이다.

STEP 01
페인 발견
"레퍼런스 사진 → 위치"를 매번 손으로 검색하는 비효율을 페인으로 정의.
STEP 02
수동 해결 1회
觀音堂 사진 한 장을 한자 OCR + 검색 + 지도 + 핀터레스트로 끝까지 해결.
STEP 03
결과물 HTML화
매거진 톤 보고서로 정리. 이게 다음 작가도 받아볼 수 있는 형태여야 한다고 판단.
STEP 04
워크플로우 추출
방금 손으로 한 일을 7단계로 분해. 각 단계의 의사결정 기준 명시.
STEP 05
SKILL.md 작성
트리거·인풋·각 단계·디자인 톤·하드룰을 한 파일에 압축. 예시 리포트 첨부.
STEP 06
GitHub Pages 배포
컬렉션 + 스킬 본체를 같은 저장소에. 누구나 받을 수 있는 형태로 공개.
02

스킬이 자동화하는 7단계

What the skill does on every invocation.
DEFINED IN
SKILL.MD
01 / ALLOCATE

리포트 번호 할당

기존 번호 스캔 → 다음 NNN_{slug}/ 폴더 생성. 입력 사진 복사.

02 / IDENTIFY

위치 추정

OCR + 비주얼 시그니처 + EXIF + WebSearch. 신호 3개 이상 교차 검증.

03 / VERIFY

현재 상태 확인

최근 6개월 리뷰 + 폐업/리모델링 점검. 예전 사진과 지금이 같은지.

04 / CONDITIONS

촬영 조건 정리

입장료·허가·골든아워·혼잡 시간. 답사 전 알아야 할 항목 압축.

05 / MATCH

비주얼 매칭

핀터레스트 related-pins 알고리즘 활용. Playwright 무로그인 + curl 다운로드.

06 / RENDER

리포트 HTML 생성

매거진 톤 템플릿에 채워서 출력. 출처 [N] 각주는 필수.

07 / PUBLISH

컬렉션 업데이트

index에 카드 추가 → git push → GitHub Pages 자동 빌드. 30초 안에 라이브.

— / GUARANTEES

하드 룰

기존 파일 덮어쓰기 금지, 출처 누락 금지, 사진 placeholder 금지, 신뢰도 솔직히.

03

내 Claude Code에 설치하기

Install the skill into your own Claude Code.
REQUIRES
CLAUDE CODE

전제 조건  ·  Claude Code CLI 설치 완료 (claude.com/claude-code), 그리고 핀터레스트 매칭을 쓰려면 Playwright MCP 서버가 연결되어 있어야 한다. 두 가지가 갖춰져 있다면 아래 한 줄로 끝난다.

STEP 01 — CLONE

이 저장소를 받는다

SNAP-SCOUT의 컬렉션과 스킬 본체가 함께 들어있다. 임시 폴더 어디든 OK.

$ git clone https://github.com/tryoutmarket/tm-lab-snap-scout.git $ cd tm-lab-snap-scout
STEP 02 — INSTALL

skill/ 폴더를 Claude Code skills 경로로 복사

이 한 줄이면 Claude Code가 다음부터 snap-scout이라는 스킬로 인식한다.

$ mkdir -p ~/.claude/skills/snap-scout $ cp -r skill/* ~/.claude/skills/snap-scout/
STEP 03 — VERIFY

제대로 인식됐는지 확인

Claude Code를 다시 띄우고 아래처럼 물어보면, 시스템이 "snap-scout" 스킬을 알고 있다고 표시한다.

$ claude # 안에서: > 지금 쓸 수 있는 스킬 목록 보여줘
STEP 04 — INVOKE

사진을 던지면서 호출

로컬 경로든, URL이든, 핀터레스트 핀 링크든 OK. 한국어/영어/일본어 어느 쪽이든 트리거된다.

> 이 사진 snap-scout으로 정찰해줘 [photo.jpg] # 또는: > snap-scout this: https://pinterest.com/pin/... # 또는 그냥: > 여기 어디인지 알려줘 [photo.jpg]
04

트리거와 인풋

When the skill activates — and what it expects.
SEE SKILL.MD
§ 1 — 2

이런 말에 켜진다

  • [KO] "이 사진 snap-scout 돌려줘" 사진 + 직접 호출
  • [KO] "여기 어디야? 답사 리포트 만들어줘" 의도 기반 호출
  • [EN] "snap-scout this pin" 핀터레스트 URL 직접 입력
  • [KO] "스냅스카웃 / 정찰해줘 / 이거 분석해줘" 짧은 명령형

이런 건 트리거 안 됨

  • [—] "이거 무슨 꽃이야?" 사진 속 사물 식별은 별개
  • [—] "역검색해줘 (general reverse search)" 출사 의도 없는 검색은 미적용
  • [—] "이 사진 톤 따라 보정해줘" 후반 작업은 스킬 영역 밖
  • [—] "이 보드 전체 분석해줘" v2 로드맵 — 아직 미구현

한 명의 작가에서
모든 작가로.

이 스킬은 한 사람의 페인에서 시작했지만, 페인 자체가 보편적이라 그대로 다른 작가에게 옮겨진다. 누구든 받아서 쓰고, 자기 워크플로우에 맞게 SKILL.md를 고치면 된다.

코드 한 줄도 직접 짤 필요 없이, Claude Code가 SKILL.md를 읽고 흐름을 그대로 실행한다. 기여나 다음 페인 발견이 있다면 저장소 이슈로.
